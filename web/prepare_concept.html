什么是Mod？

在《我的世界》里，mod（组件）为游戏带来了无限拓展可能。它主要由游戏地图、功能玩法、材质光影、皮肤和其他不常用的部分组成。这些丰富多样的mod，让每个玩家都能在《我的世界》中打造出属于自己的独特天地。

皮肤其实就是一张png贴图，搜索引擎搜索我的世界皮肤制作即可找到很多大神分享的教程。

功能玩法与材质光影都是通过附加包来制作。

游戏地图一般是一个搭建好的游戏存档，但是还可以添加附加包来实现更丰富的玩法。

什么是附加包？

附加包（Addon）是《我的世界》基岩版中与Java版模组相对应的内容扩展形式，是基岩版对mod的一种支持方式。

与Java版不同的是，Java版模组侧重于修改游戏本身，而附加包则遵循微软提供的开发规范，通过添加新内容来扩展游戏。

附加包分为两种类型：资源包（resource_pack）和行为包（behavior_pack）。两者可以独立运行，但通常配合使用。

附加包可以做什么

附加包允许我们通过修改或移除现有内容以及添加自定义内容来改变Minecraft的游戏体验。

它们功能强大，可以创建自定义实体、物品、方块，以及自定义战利品表和合成配方等。你的想象力就是唯一的限制！

附加包主要使用结构化数据格式JSON编写。本质上，附加包是一系列JSON文件、图像和声音文件的集合，通过这些文件以某种方式修改或增强游戏内容。

理解JSON

JSON是一种简单的文本文件格式，既方便人类阅读又便于计算机解析。

基岩版使用.json文件作为附加包的编程语言，因此您需要扎实掌握JSON的读写能力！

编写JSON时最关键的是必须确保文件完全无错误，否则整个文件将无法正常工作。即使一个错误的字符或多余逗号都会导致解析失败。因此，编写有效JSON至关重要。

JSON中使用多种格式表示数据，每种格式对应特定数据类型：

名称 示例 说明 游戏内代表
字符串 "Hello!" 文本字符。必须用引号""包裹。 游戏内一般用来表示特定的字段
整型 15 整数。无需引号 游戏内一般用来定义数值
浮点型 1.2 小数。无需引号 游戏内一般用来定义数值
布尔值 true 只能是true或false。无需引号 游戏内一般用来定义某个功能是否开启

除了上述类型，还有两种特殊数据结构用于嵌套数据：

  • 数组使用方括号[]表示，本质是有序列表。方括号内可以是任意数据类型，元素间用逗号分隔。

  • 对象使用花括号{}表示，包含键值对集合。键名称为key，对应数据称为value。键值对的格式为"<key>": <value>，注意键名必须用引号包裹且后接冒号。

    键值对之间需要用逗号分隔。我们称对象内的键值对为它的子元素或内部元素。

除了字符串的引号内的文本内容可以写中文，其余部分，包括字符串的引号都必须是半角英文，以下为具体的写法：

JSON 写法 
{
    "my_string": "你好", // 这是字符串
    "my_int": 15, // 这是整型
    "my_float": 1.2, // 这是浮点型
    "my_bool": true, // 这是布尔值
    "my_ints": [1, 2, 3], // 这是数组，数组里是整型
    "my_strings": ["红", "蓝"], // 这也是数组，数组里是字符串
    "my_mixed": ["红", 1, false], // 这也是数组，数组里包含了字符串、整型和布尔值
    "my_dict": { // 这是对象，对象里可以是任意数据类型
        "my_ints": [1, 2, 3],
        "my_bool": true
    }
}

在游戏中，JSON用于为一切自定义内容提供数据。所有使用这种方式自定义的内容都被称为数据驱动。

我们可以在游戏内用数据驱动的方式，定义物品、方块、实体、动画、UI等各类内容，几乎涵盖了大部分可自定义的内容。

值得注意的是，数据驱动的方式只能定义一个对象的初始状态。如果要在游戏运行时动态地更改内容，你还需要脚本的辅助。

如果你仔细查看过附加包的文件结构，你会发现有些.JSON文件位于名字包含beh的文件夹下，而有些位于名字包含res的文件夹下。前者被称为行为包，后者被称为资源包。

不管在行为包还是资源包中，.JSON文件一般都是成组地被分在不同的子文件夹下。

比如：战利品表就都位于loot_tables文件夹内，而网易的自定义方块则位于netease_blocks文件夹下。
